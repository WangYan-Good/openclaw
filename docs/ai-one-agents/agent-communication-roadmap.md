## 📐 沟通规则

### 规则 1: 层级关系

| 层级 | Agents | 沟通权限 |
|------|--------|---------|
| **L0** | AI-Secretary | 可唤醒所有 agents |
| **L1** | CTO, COO, CPO, Auditor | 可唤醒本部门 L2 + Peer 沟通 |
| **L2** | 所有执行层 agents | 仅平级沟通，紧急可唤醒上级 |

### 规则 2: 沟通方向

```
┌─────────────────────────────────────────────────────────────┐
│                    沟通方向示意图                            │
└─────────────────────────────────────────────────────────────┘

CEO ←→ Secretary  (双向，随时)
   │
   ▼
Secretary ←→ L1 (双向，随时)
   │
   ▼
L1 ←→ L1 Peer  (双向，sessions_send)
   │
   ▼
L1 → L2 (唤醒，sessions_spawn)
   │
   ▼
L2 ←→ L2 Peer (双向，sessions_send)
   │
   ▼
L2 → L1 (仅紧急，需【紧急】前缀)
```

### 规则 3: 跨部门沟通

```
跨部门沟通必须通过 Peer 连接点:

软件服务部 ↔ 市场金融部
    CTO ←→ COO (部长级中转)

软件服务部 ↔ 审计中心
    CTO ←→ Auditor (审计关系，Auditor 可直接审查)

市场金融部 ↔ 审计中心
    COO ←→ Auditor (审计关系，Auditor 可直接审查)

产品战略 ↔ 软件服务部
    CPO ←→ CTO (技术路线协调)

产品战略 ↔ 市场金融部
    CPO ←→ COO (产品运营协调)
```

## 📞 沟通方式详解

### 1. sessions_spawn (唤醒)

**适用场景**: 上级唤醒下级，分配新任务

```javascript
// AI-Secretary → AI-CTO
sessions_spawn({
  agentId: "cto",
  task: "开发 AI-One Phase 1 热更新系统",
  mode: "session",
  thread: true,
  timeoutSeconds: 7200
});

// AI-CTO → AI-Backend
sessions_spawn({
  agentId: "backend",
  task: "实现用户认证模块",
  mode: "session",
  thread: true,
  timeoutSeconds: 3600
});
```

### 2. sessions_send (发送消息)

**适用场景**: 平级沟通，或向已存在会话发送消息

```javascript
// AI-Backend → AI-Frontend (平级)
sessions_send({
  sessionKey: "agent:frontend:main",
  message: "API 接口已更新，请同步更新前端调用"
});

// AI-PM → AI-CTO (汇报)
sessions_send({
  sessionKey: "agent:cto:main",
  message: "PRD 文档已完成，请技术评审"
});
```

### 3. openclaw agent CLI (直接唤醒)

**适用场景**: 从命令行直接唤醒任意 agent

```bash
# AI-Secretary 唤醒 AI-CTO
openclaw agent --agent cto --message "开始 Phase 1 开发"

# AI-Secretary 唤醒 AI-Finance
openclaw agent --agent finance --message "编制 Q2 预算"
```

### 4. inbox 文件 (异步沟通)

**适用场景**: 异步任务分配，文档传递

```bash
# 写入 inbox 文件
echo "请审查以下代码" > ~/.openclaw/workspace-backend/inbox/review-request.md

# Agent 定期查看 inbox
```

## 🚨 紧急情况处理

### 紧急级别定义

| 级别 | 前缀 | 适用场景 | 处理方式 |
|------|------|---------|---------|
| **普通** | (无) | 日常任务 | 正常流程 |
| **紧急** | 【紧急】| 严重 bug、生产问题 | 直接唤醒上级 |
| **极紧急** | 【极紧急】| 系统宕机、安全漏洞 | 越级上报 CEO |

### 紧急上报流程

```
Level 2 (执行层)
  │
  ▼ 发现紧急情况
  │
  ├──► 直接上级 (L1) ──30 分钟无响应──► 越级上报
  │      │
  │      ▼
  │   AI-Secretary
  │      │
  │      ▼
  │     CEO
  │
  └──► 【极紧急】AI-Secretary (直接)
         │
         ▼
        CEO
```

## 🎯 沟通最佳实践

### ✅ 推荐做法

1. **平级直接沟通** - 不要通过上级中转
2. **紧急加前缀** - 【紧急】/【极紧急】
3. **跨部门找 Peer** - 通过部长级连接点
4. **定期汇报** - L2 → L1 每日，L1 → L0 每日
5. **文档留痕** - 重要沟通写入 inbox

### ❌ 避免做法

1. **越级汇报** - 除非极紧急
2. **跨部门直连** - 必须通过 Peer
3. **无响应沉默** - 收到消息需确认
4. **重复唤醒** - 等待响应再跟进
