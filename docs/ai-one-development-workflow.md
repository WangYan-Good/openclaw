```
└─────────────────────────────────────────────────────────┘
```

**审查清单**:

### Reviewer-Clean (整洁代码)
- [ ] 命名清晰易懂
- [ ] 函数单一职责
- [ ] 避免重复代码 (DRY)
- [ ] 注释解释"为什么"而非"是什么"
- [ ] 代码格式规范
- [ ] 遵循 SOLID 原则

### Reviewer-Security (安全 + 性能)
- [ ] 无 SQL 注入风险
- [ ] 无 XSS/CSRF 漏洞
- [ ] 敏感数据加密
- [ ] 认证授权正确
- [ ] 时间复杂度合理
- [ ] 内存使用合理
- [ ] 无内存泄漏

**交付物**:
| 文档 | 说明 | 负责人 |
|------|------|--------|
| 审查意见 | PR Comments | Reviewers |
| 审查报告 | 审查总结 | Reviewer-Security |

**审批节点**:
- ✅ **双 Reviewer 批准** → 进入 Phase 3
- ❌ **任一 Reviewer 驳回** → 返回修改
